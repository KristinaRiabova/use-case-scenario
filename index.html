Name
Game Tracker and Quest Manager
Short Description
Game Tracker and Quest Manager is an app for players who want to organize their games and track quests. It helps users see their progress and manage their gaming activity in one place.

Use Case

Use Case Title
Complete a Quest
Goal
The user wants to mark a quest in a game as completed and update game progress.
Trigger
The user opens the game page and chooses a quest to complete.

Actors

Primary Actor
Registered User
Secondary Actors
System Database

Preconditions

  • The user has an account and is logged in.
  • The game is already added to the user’s library.
  • The selected game has at least one quest.
  • The quest is not completed yet.

Postconditions

Success

  • The quest is marked as completed.
  • The game progress is updated.
  • The new information is saved in the database.

Failure

  • The quest remains incomplete.
  • No progress changes are saved.

Main Success Scenario

1

The user logs into the application.

2

The user opens their game library.

3

The user selects a game from the list.

4

The system shows the game details and the list of quests.

5

The user finds a quest and clicks the “Complete Quest” button.

6

The system checks if the quest belongs to this game and if it is still active.

7

The system marks the quest as completed.

8

The system updates the game progress percentage.

9

The system saves the new data in the database.

10

The system shows a message: “Quest completed successfully.”

Alternative Scenarios

Functional Requirements

  • The system must allow a logged-in user to view their game library.
  • The system must show quests for each game.
  • The system must allow the user to mark a quest as completed.
  • The system must validate quest status before updating it.
  • The system must update game progress after quest completion.
  • The system must save all changes in the database.
  • The system must show error messages when an action cannot be completed.
